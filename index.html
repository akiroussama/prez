Calibration — Le Sharpening

AVANT — Le Probleme

XGBoost (vert) groupe a ~25% → seuil 80% jamais atteint

APRES — Sharpening

XGBoost calibre → decisions nettes au-dela du seuil

p_sharp = p³ / Σ(p³)

Concentre la probabilite sur la classe dominante

Resultat

Apres sharpening : le Voting est mieux calibre, ~70% d'automatisation au seuil de 80%.

TEXTE : Quel classifieur ?

Chaque choix technique resulte d'un arbitrage raisonne entre performance, complexite et contraintes.

Principe directeur

Le gain potentiel de CamemBERT (~1-2%) ne justifie pas la complexite (GPU, 200ms, boite noire). LinearSVC = meilleur compromis.

Retour d'Experience

Ce que ce projet nous a appris en tant qu'ingenieurs ML.

Ce qu'on referait autrement
  • 1. Commencer par la fusion des le debut, pas apres avoir finalise chaque pipeline separement
  • 2. Implementer le monitoring des metriques des le premier modele
  • 3. Tester l'OCR sur les images-texte avant d'investir dans le fine-tuning image
Erreurs instructives
  • 1. ResNet "Phoenix" : 91% train, 65% test. Lecon : surveiller le gap train/val
  • 2. Labels desalignes : 3 referentiels incompatibles. Lecon : standardiser le mapping
  • 3. XGBoost non calibre : probabilites "molles" diluaient le vote
